## 一、需求 (Requirements)

### 1.1 目标用户

**普通 KOC（独立内容创作者）**，月粉丝 1k–10w 量级，单兵作战，无团队/MCN 资源支持。

| 维度 | 描述 |
|---|---|
| **年龄段** | 22–35 岁为主（核心 25–32 岁），覆盖 Z 世代尾部到 90 后中坚 |
| **职业身份** | 上班族 / 自由职业 / 在校生 / 宝妈，把短视频作为副业或转型路径 |
| **行为特征** | 每天刷短视频 ≥ 90 分钟（重度内容消费者）；月均购买 1-2 次内容相关付费品（剪映会员、配音工具、模板库等）；社交平台同时活跃 3 个以上（抖音 + 小红书 + 微信视频号 / B 站） |
| **每日可投入** | 1–3 小时，难以应付完整的"选题—脚本—包装—互动"流程 |
| **缺乏的能力** | 编导思维、爆款拆解经验、SEO 知识、评论运营策略 |
| **已具备的资源** | 一台笔记本 / 一部手机 / 对自身领域的真实经验 |
| **预算** | 极低，对 SaaS 月费 > 99 元已敏感 |

> **市场规模**（根据星子文化《2025 KOC 达人推广行业白皮书》）：全网 KOC 活跃达人**突破 1000 万人**，分布在抖音、快手、小红书、知乎、B 站等多平台。其中年收入区间在 3000-8000 元 / 月的中长尾创作者占绝对多数——**这是 Seecript 的核心目标用户群**。

### 1.2 用户面对的三大痛点（含数据 + 规则 + 故事三轨佐证）

#### 痛点 ① · 选题/创作焦虑

> 看到爆款只能"凭感觉模仿"，模仿出来的常常是表层而不是结构。

| 佐证类型 | 内容 |
|---|---|
| **行业数据** | 据新榜《2024 内容创作者生态报告》，**抖音博主涨粉千万的平均用时从 2018 年的 54 天延长至 2023 年的 601 天，需要花费的时间是 5 年前的 10 倍以上**；2024 年 6 月头部万账号涨粉量**同比下滑 3.8%**——爆款门槛在持续抬升，"凭感觉模仿"的成功率越来越低 |
| **行业数据** | 单条视频制作时间分层显著：**新人需 4-5 小时**、**成熟期 1 小时**、**头部 MCN 标准化压到 45 分钟**（《抖音创作者效率报告 2025》）。中长尾 KOC 卡在第一阶段，每条视频耗时是头部 MCN 的 5-7 倍 |
| **平台规则** | 抖音算法把**完播率作为最高权重指标（占 40%）**，且要求"前 3 秒钩住人"。Hook 设计能力不足 = 视频在算法层就被淘汰，根本进不到内容质量评比阶段 |
| **案例场景** | 周六上午 9 点，小李刷到一条 30 万赞的反内卷视频决定模仿。9:30 开始抄脚本，10:30 卡在"开头怎么钩住人"——抄字面是抄了，但发出去 30 万赞那种"反差感"找不回来。这就是痛点 ① 的真实瞬间——**模仿了表层，没拿到结构** |

#### 痛点 ② · 包装/分发缺位

> 写完脚本就没力气做标题/简介/标签了，长尾搜索流量持续被埋。

| 佐证类型 | 内容 |
|---|---|
| **行业数据** | 抖音官方 2024 创作者大会披露：**抖音总流量中 22% 与用户搜索相关，76% 的创作者因搜索获得了更多流量**；80% 的用户每月会用抖音搜索，搜索规模同比增长 51%。**搜索流量被搁置 = 一年损失四分之一以上的曝光** |
| **行业数据** | 同条视频，**标题从 28 字精简到 19 字（命中算法甜蜜点 15-20 字），完播率提升 47%**（《2025 短视频算法解构白皮书》）。标题质量直接决定推流上限 |
| **平台规则** | 抖音算法**主关键词必须在标题前 5-6 字内出现**才能被精准识别；标签布局遵循 5-3-2 法则——50% 核心业务词、30% 场景化长尾词、20% 热点借势词；**单条视频用 5-8 个精准标签的完播率比堆砌高 37%** |
| **平台规则** | **参与热门挑战的视频平均播放量是普通视频的 4.7 倍**，互动率提升至 6.2%——但用户必须持续监控话题热度才能赶上窗口期 |
| **案例场景** | 写到第 3 个小时，小李把脚本剪好导进剪映，准备发布——这时候标题框还是空的。他随手敲了个"打工人都来看看"就发了，结果两天后再回看，那条视频的搜索曝光为零。这就是痛点 ② 的真实瞬间——**写完脚本人已经累了，包装这一步直接被跳过** |

#### 痛点 ③ · 互动/数据吃力

> 评论越多越焦虑，分不清高价值评论与灌水，回复语气也常出错。

| 佐证类型 | 内容 |
|---|---|
| **行业数据** | **每增加 1 条 30 字以上的长评论，视频停留时长平均提升 8 秒**——长评论既影响推流权重，又难筛选；普通创作者面对几十条评论时往往只能挑一两条"看着顺眼"的回，价值评论被淹没 |
| **行业数据** | **及时回复评论的创作者粉丝留存率高出 35%、互动率提升 50%**（《2025 视频号评论运营白皮书》）；新视频发布后**黄金 6 小时**内是互动关键期，快速反馈能获 15-30% 流量加权——但单兵创作者很难在黄金 6 小时内同时做好脚本与评论运营 |
| **平台规则** | **当视频评论突破 200 条时触发次级流量池**，但真正影响权重的是"有效评论率"（15 字以上实质性内容）和创作者回复率。回复语气不当（敏感场处理不好、套话过多）反而会让评论区变冷，触发反向算法降权 |
| **案例场景** | 视频发布 24 小时，评论 50 条。小李一边摸鱼回复一边后悔——"早知道那条争议评论应该用幽默语气，结果我用了专业解读，把人怼到了下次不来了"。这就是痛点 ③ 的真实瞬间——**评论运营没有标准动作，全靠当下情绪** |

> **小结**：三大痛点都不是"用户偷懒"，而是**单兵创作者面对 MCN 团队化分工时的能力差距 + 时间约束 + 平台算法门槛三重挤压**的结果。

### 1.3 产品目标

| 类别 | 目标 |
|---|---|
| **效率** | 单兵创作者从 0 → 一份带人设 / 脚本 / 发布元数据的产物，**用时 ≤ 30 分钟** |
| **质量** | AI 输出结构稳定（Hook + 多段 Body + CTA 时间轴对齐）；选项可朗读、不照抄原台词 |
| **可控** | 引导式问答**严格 ≤ 3 轮**，杜绝"AI 越问越乱、用户决策疲劳" |
| **低成本** | 后端单实例（4G 内存）、前端纯静态、视频抽轨在浏览器内完成；月度算力成本 ≤ 100 元 / 单租户 |

## 二、核心功能 (Core Features)

### 2.0 产品概述（一句话讲清楚）

> **Seecript 是一个为单兵创作者设计的 AI 副驾产品——它把"对标爆款 → 拆解结构 → 生成原创脚本 → **分镜短视频素材** → 包装发布元数据 → 分拣评论运营"六个环节用一条工作流串起来，让一个人在 30 分钟内做完原本 MCN 团队需要 3 小时才能完成的内容生产闭环。**

Seecript 由 **5** 个对外功能模块组成，加 1 个工作台做枢纽。下文每个模块按"它解决什么痛点 → 用户看到什么 → 关键交互"三段叙述。

### 2.3 模块二 · 爆款拆解 + 脚本引擎（`feature-1.html` · 6 步工作流）

**解决痛点**：①「凭感觉模仿爆款，模仿不到结构」。

#### 第 0 步 · 选定本次使用的人设

不让用户跳过——后续生成的脚本要"贴合什么人设"必须先确定。

![第 0 步 · 选定本次使用的人设](docs/screenshots/s-05.png)

#### 第 1 步 · 输入对标内容

两条路径二选一——上传视频音频，或者直接粘台词。

![第 1 步 · 输入对标内容（视频/文本二选一）](docs/screenshots/s-06.png)

#### 输入约束（台词长度 · 上传建议 · v0.10.2）

| 路径 | 契约 |
|---|---|
| **粘贴台词** | `POST /api/skeleton/extract` 等请求的 `transcript` 与后端 **`TRANSCRIPT_MAX_CHARS`（默认 50000 字）** 对齐；问答 / 脚本链路可选台词字段同上限。超长请删减或分段拆解。 |
| **上传音视频 → ASR** | **建议单次素材约 1 分钟**（更长可传：解析更久、文件更大，且抽取后音频须 ≤ **25MB** 方可调用豆包极速版）。 |

#### 第 2 步 · AI 拆解骨架

拆解后用户拿到 4 类信息：Hook 卡（6 种钩子策略枚举）、Body 节奏卡（3-6 段含情绪标签）、CTA 卡（5 种行动呼吁枚举）、可迁移模板。

![第 2 步 · 拆解骨架完成](docs/screenshots/s-07.png)

#### 第 3 步 · 引导式问答（含 v0.8 新增 brief 创作要求）

本步分两个阶段：

**阶段 A · 创作要求（brief）表单** —— v0.8 新增

3 组 chip 互斥单选 + 1 个 200 字自由补充框。默认值：30s · 紧凑 · 中性。点「用上面要求开始 3 轮单选问答 →」 → **brief 立即锁定**。

![阶段 A · 创作要求表单](docs/screenshots/s-08.png)

**阶段 B · 3 轮单选问答**

| 轮 | 主题 | 用户在做的决策 |
|---|---|---|
| 1 | Hook 角度 | 视频前 3 秒用什么钩子 |
| 2 | Body 切入 | 主体节奏怎么展开 |
| 3 | CTA 风格 | 行动呼吁怎么写 |

每题用户单选一项，**全程不允许自由文本输入**。满 3 轮后系统**强制**结束问答，自动进入第 4 步生成脚本。

![阶段 B · 3 轮单选问答](docs/screenshots/s-09.png)

#### 第 4 步 · 原创分镜脚本

脚本输出 = Hook 段 + N 个分镜段（与骨架的 Body 等长） + CTA 收尾段。每段含时间区间、画面提示、口播台词。

![第 4 步 · 原创分镜脚本输出](docs/screenshots/s-10.png)

第 4 步面板提供 CTA「**→ 分镜素材生成**」：将最近一次成功的脚本对象写入 `sessionStorage`，跳转 `feature-5.html` 作为默认「素材描述」并支持按 Hook / 分镜 / CTA **单选**生成竖屏素材短片（详见 §2.6）。

### 2.5 模块四 · 评论分拣（`feature-4.html`）

**解决痛点**：③「评论越多越焦虑、分不清价值、回复语气出错」。

![评论分拣 · 三级输出（高/中/灌水）](docs/screenshots/s-14.png)

#### 输出结构

- **高价值** — 上限 5 条，分类含"干货提问 / 争议探讨 / 高互动潜力 / 下期选题 / 敏感场"。每条配 3 种语气回复草案：**专业解读 / 幽默调侃 / 共情安抚**，每条 ≤ 80 字
- **中价值** — 仅标分类（中价值 / 下期选题）
- **灌水** — 仅显示总数（如"已识别灌水 23 条，不展示"）

### 2.6 模块五 · 分镜素材生成（`feature-5.html` · v0.9 起）

**解决痛点**：①③ 交织——有脚本但**不愿出镜 / 缺素材**，需要快速拿到 **可剪辑的竖屏镜头片段**，而非停留在纯文本。

#### 产品定位（v3.3 明确）

- 单次提交生成的是 **一段约 5～10 秒的竖屏短视频素材**（默认智谱 **CogVideoX-3**，服务端可配置时长；可选 **cogvideox-2** 低价短片段）。
- **不是**「一键成片解说视频」；完整口播成片由用户在 **剪映 / Premiere / 达芬奇** 等工具中 **导入多条素材 + 配音 + 字幕** 合成。
- 有结构化脚本（Hook / 各分镜 / CTA）时：页面 **先列分镜单选**，再同步「素材描述（提示词）」；可多次生成、多次下载 **多条 MP4** 作为 B-roll 或镜头预览。

#### 关键交互

| 操作 | 用户感受 |
|---|---|
| 从 feature-1 第 4 步点「→ 分镜素材生成」 | 带入脚本；若有分镜列表则默认选中第一条并填充描述 |
| 点「生成分镜素材」 | 扣费一次（失败不扣，依智谱结算规则）；轮询直至成功或失败 |
| 成功页「下载 MP4」 | 本地保存素材；用户再到剪辑软件与完整工程对齐时间轴 |

### 3.1 7 个 AI 干预点

| 1 | 人设生成 | feature-2 | 把"3 段背景"转化为 3 个差异化人设方案 |

| 3 | 引导式问答 | feature-1 第 3 步 | 把"骨架 + brief"转化为 3 轮单选题，让用户做关键创作决策 |

加上一个**前置非 LLM 干预**——视频音频转写（feature-1 第 1 步路径 A，使用专门的录音文件识别服务，1-5 秒返回）。

### 3.2 去掉 AI 这个产品还能成立吗？

> **答案：不能。本产品 7 个核心功能里 7 个都是 AI 推理；去掉 AI 就退化为一个空白工作台。**

下表逐一对照"AI ↔ 痛点"的强绑定关系：

| 干预点 | 它具体做什么生成式工作 | 没有 AI 时的退化形态 | 用户痛点是否还能解决 |
|---|---|---|---|
| 人设生成 | 综合三段背景 → 创造性输出 3 个有差异化的人设方案（含起号路径、对标账号、变现预判） | 退化为一个空白表单，让用户自己填 | ❌ 用户不知道写什么——这正是痛点 ① |
| 爆款拆解 | 从一段非结构化台词中**抽取叙事结构**（Hook 6 策略 + Body 多段 + CTA 5 策略）+ 总结可迁移模板 | 退化为人工拆解教程列表 | ❌ 拆解能力本身就是用户缺乏的——这是痛点 ① 的根因 |
| 引导式问答 | 基于骨架 + 用户人设 + brief，**生成具体可朗读的选项句子**（不是抽象问题） | 退化为静态问卷模板，AI 给不出"贴合此爆款 + 此人设 + 此 brief"的具体选项 | ❌ 选项必须是 AI 当场推理才能"具体可朗读"——抽象问卷反而加重决策疲劳 |
| 原创脚本 | 把 4 个输入维度（骨架 / 答案 / 人设 / brief）融合 → **生成结构对齐 + 时长精准的分镜脚本** | 退化为脚本模板填空 | ❌ 模板填空就是用户最初想做的事——他做不好才来用产品 |
| 标题/简介/标签 | 从脚本生成 5 个**带类型差异**的标题（反常识/数字/身份/痛点/悬念）+ 算法适配的简介 + 三类标签矩阵 | 退化为 SEO 规则文档 | ❌ 用户已经累了——SEO 文档让他自己看反而是反人性 |
| 评论分拣 | 阅读 N 条评论 → 三级分级 + **针对每条高价值评论生成 3 种语气的回复草案** | 退化为关键词过滤器 | ❌ 关键词过滤分不出"干货提问"和"争议探讨"，更生不出回复 |
| 分镜素材（文生视频） | 基于提示词 / 选定分镜 → **生成数秒级竖屏视频像素素材**（上游 T2V API） | 退化为用户自己找素材库或自拍 | ❌ 出镜门槛与制作成本仍在——素材仍是刚需 |

### 3.3 AI 生成能力的强大之处具体体现在哪？

Seecript 不只是"调一下 LLM 包个壳"——以下是它**只有靠强生成式 AI 才能做到**的具体能力点：

#### 能力一 · 跨域抽象与结构化（爆款拆解）

模型需要从一段几百字到几千字的非结构化台词中：
- 识别**叙事节奏断点**（哪里是 Hook、哪里转折、哪里是 CTA）
- 抽象出**情绪曲线标签**（好奇/共鸣/反转/认同）
- 生成"剥掉具体内容、保留结构"的**可迁移模板**

这要求模型同时具备「文本理解」「叙事学知识」「抽象建模」三种能力——传统 NLP 抽取式方法无法做到。

#### 能力二 · 多条件融合的可控生成（脚本引擎）

第 4 步生成脚本时，AI 要**同时满足 5 个硬约束**：
1. scenes 数量 = 骨架 body 段数（结构对齐）
2. 每段 timestamp = 骨架对应段（时间轴对齐）
3. hook_narration / 中段 / cta_narration **分别落地** 3 个用户答案
4. 全文风格贴合人设
5. 总字数按朗读速度对齐 brief 时长（约 3.5 字/秒）

5 个约束**单独不难，叠加起来对模型推理能力是真考验**——这是 DeepSeek-V3 / R1 这一代模型才能稳定做到的事。

#### 能力三 · 结构化分级 + 多语气改写（评论分拣）

模型要在一次推理内：
- 读 N 条评论并**主观判断价值层级**（干货提问 vs 争议探讨 vs 灌水）
- **同一条评论生成 3 种语气**的回复草案（专业 / 幽默 / 共情），且每种语气**风格可识别地不同**
- 自动剔除敏感场的不当回复

这本质是"理解 + 评判 + 三轮改写"的复合任务，传统规则引擎做不到。

#### 能力四 · 引导式收敛对话（3 轮问答）

模型要根据已答历史 + brief + 骨架，**生成"具体可朗读、之间有差异、贴合人设"的 3-4 个选项**。这需要：
- 理解用户已经做出的决策（不重复出题）
- 在创作维度上保持差异（不同选项之间不能"同义不同字"）
- 选项是"立刻可拍"的具体句子，而不是"你可以怎样怎样"的开放式问题

这种"生成式对话设计"恰恰是大模型最擅长、规则引擎最不擅长的领域。

> **结论**：Seecript 不是把 AI 作为辅助工具，而是把 AI 作为**核心生产引擎**——产品的每个价值瞬间都需要 AI 的生成能力支撑。

### 3.4 所用模型 / API / 平台 + AI 工作流程

#### 模型与平台清单

| 用途 | 模型 / 服务 | 平台 / API | 调用方式 |
|---|---|---|---|
| **大语言模型** | DeepSeek-V3 / R1 | DeepSeek 开放平台（OpenAI 兼容协议） | HTTP `/chat/completions`，单实例每次请求独立 client |
| **录音文件识别** | 豆包大模型录音文件识别 · 极速版 | 火山引擎（资源 ID `volc.bigasr.auc_turbo`） | 单次 HTTP POST，base64 内联音频，1-5 秒返回 |
| **视频抽轨** | `ffmpeg.wasm 0.12.x` | **完全在用户浏览器内运行** | 0 服务器 CPU；vendored `/vendor/ffmpeg/` 解决 Worker 同源约束 |
| **离线 mock** | 内置 `MockLLMClient` / `MockASRClient` | 无外部依赖 | 通过 system prompt 内独占字段名做 fingerprint 路由，零外部依赖跑通完整 UI |

#### AI 在产品中的工作流程

```
用户输入 → 浏览器抽轨/校对 → POST /api/* (FastAPI 路由层)
                                  ↓
                        services/prompts/*.py 注入 system 指令
                                  ↓
                        services/llm_client.py 抽象层（DIP）
                                  ↓
                        DeepSeek API (OpenAI 兼容) / Mock
                                  ↓
                        complete_json() 解析（含一次 JSON 失败重试）
                                  ↓
                        Pydantic v2 严格校验所有字段（Schema 终判）
                                  ↓
                        路由层硬约束（如"3 轮强收敛"由后端而非 AI 决定）
                                  ↓
                        返回结构化 JSON → 前端按模板渲染
```

**关键架构选择**：
- 业务路由只依赖抽象 `LLMClient`，新增模型只需写新子类（OCP）
- 6 个干预点共享同一个 `complete_json` 入口——异常在抽象层收敛，业务层不必各自防御（DIP）
- mock 模式 fingerprint 路由确保离线演示与 CI 都能跑通

> 详细实现细节（每个干预点的 prompt 关键约束、三层兜底矩阵、状态码映射）见 `docs/AI-DESIGN.md`。

### 3.5 AI 输出的可控性

LLM 是不可信的——它会偶尔漂移、忘记指令、输出畸形 JSON。Seecript 用三层兜底来保证产物可控：

| 层 | 怎么做 |
|---|---|
| **指令层** | 每个干预点都有一份独立的 system 指令，写明角色、任务、输出格式、硬性规则 |
| **业务层** | 关键约束在后端代码里再拦一道（如"3 轮强制收敛"由后端而非 AI 决定） |
| **校验层** | AI 返回的所有 JSON 都做严格校验，畸形则拒绝 |

### 3.6 关键设计选择

| 设计选择 | 为什么 |
|---|---|
| 引导式问答只让用户单选，不开放自由输入 | 早期版本提供过自由输入，实测 90% 对话不收敛、产物比纯选项版本质量更低。"给具体选项"反而比"问开放式问题"更尊重创作者的时间 |
| 引导式问答严格 ≤ 3 轮 | 用户决策疲劳从第 4 题开始陡升。3 轮够覆盖 Hook / Body / CTA 三个关键决策点 |
| AI 不直接照抄原台词 | 用户来这里是要"原创可发布"的产物，不是再看一遍原视频 |

### 3.7 文生视频接入策略（v0.9 新增）

#### 为什么要做？

Seecript v0.8 把"文本生产"做到了端到端：从人设方案到原创脚本到标题元数据。但调研中我们注意到目标用户里有一类人**特别明确地拒绝出镜**——上班族、宝妈、内向技术人员；他们能写脚本但不愿、不便、不敢真人拍摄。这导致脚本停留在硬盘里，转化不出实际作品。

**第 7 个干预点要解决的痛点**：让这类用户能用 AI **直接生成可发布的视频素材片段**，不再卡在"我有脚本但没法拍"。

#### 接入路径选择 · 5 个候选评估

| 路径 | 形态 | 单条成本 | 实施难度 | 适配 KOC 场景 | 评分 |
|---|---|---|---|---|---|
| A · 文生视频（T2V） | 纯文本 → 视频片段 | 0.5-2 元 | 低（直调 API）| ⭐⭐⭐⭐ | **9** |
| B · 图生视频（I2V） | 用户上传图片 → 动态化 | 0.5-2 元 | 中（多一步素材准备）| ⭐⭐⭐ | 6 |
| C · 数字人 / AI 出镜 | 文本驱动数字人形象 | 5-50 元 | 高（克隆 + 合成）| ⭐⭐ | 4 |
| **D · AI 解说视频组合** | **脚本 → 文生视频 + TTS 配音 + 字幕，一站式产出** | **1-3 元** | 中（多流程编排）| ⭐⭐⭐⭐⭐ | **10** |
| E · AI 剪辑助手 | 用户上传素材 → AI 剪 | 0.1-0.5 元 | 高（视频理解 + 时间码对齐）| ⭐⭐ | 5 |

**最终选择路径 D 的渐进版**：v0.9 先落地路径 A 作为基础（**文生视频片段 / 分镜素材 MP4 下载**），把架构验证完整；产品表述（v3.3）明确为 **「分镜素材 → 用户剪辑成片」**，避免与「一键解说成片」混淆；v1.x 再叠加 TTS 配音 + 字幕合成，演进到完整路径 D。这样把 5 人日的工作分两次释放，用户每个版本都能用。

#### 模型选型 · CogVideoX-2 性价比对比

调研了国内 6 家提供视频生成 API 的供应商：

| 供应商 | 模型 | 单条成本 | 个人开发者直接申请 | 商用条款 | 综合评分 |
|---|---|---|---|---|---|
| **智谱清影** | **CogVideoX-2** | **0.5 元/6 秒** | ✅ 立即开通 | ✅ 明确允许商用 | **第 1** |
| 字节即梦 | Jimeng-T2V | 1.x 元 | ⚠️ 排队审核 | ✅ 商用 | 第 3 |
| 阿里通义 | 万相 2.1 | 1-2 元 | ✅ 立即开通 | ✅ 商用 | 第 4 |
| 快手可灵 | Kling 1.0 | 5+ 元 | ⚠️ 企业认证 | ⚠️ 受限 | — |
| MiniMax | Hailuo Video | 1.x 元 | ✅ 立即开通 | ✅ 商用 | 第 5 |
| 生数科技 | Vidu Q1 | 2 元 | ✅ 立即开通 | ✅ 商用 | 第 6 |

**选择 CogVideoX-2 的关键理由**：
1. **价格**——0.5 元/条是行业最低，匹配 Seecript "月活成本 ≤ 100 元"的财务护栏
2. **接入便利**——智谱开放平台个人账户登录后即可创建 API Key，无需企业认证
3. **架构对齐**——智谱 API 与 OpenAI 兼容协议同源，复用 `httpx`，0 新增依赖
4. **生态延展**——同账号同时能开 LLM、TTS、图像，未来切到完整路径 D 不用再换供应商

#### 工程落地方式（已实施）

```
浏览器 feature-5.html                后端 FastAPI                         智谱开放平台
─────────                            ──────────                          ──────────
Prompt 输入 ─→ POST /api/t2v/submit ─→ T2VClient.submit() ─→ POST /paas/v4/videos/generations
                                      ↓ 拿 task_id
       ←── { task_id, pending } ───── 立即返回（< 2s）
       
每 5s 轮询 ─→ GET /api/t2v/query/{id} ─→ T2VClient.query() ─→ GET /paas/v4/async-result/{id}
                                                              ↓ 30s-3min 后
       ←── { status: succeeded, video_url } ──────────────────  task_status: SUCCESS
                                                                
  下载 mp4 / 重生成 / 跳标题车间
```

**关键架构选择**：
- **`T2VClient` 抽象层**——与 `LLMClient` / `ASRClient` 完全对齐，路由层只依赖抽象（DIP）；切换到通义万相、可灵只需新写子类（OCP）
- **`MockT2VClient` 时间渐进式**——8 秒后自动从 PROCESSING 翻成 SUCCESS，让前端轮询 UI 真有进度感，CI 永远不依赖外部 API
- **失败不扣费 + 任务幂等**——智谱仅对 `task_status=SUCCESS` 计费；提交失败 / 内容审核拒绝 / 网络抖动均零成本
- **任务态完全在智谱侧**——Seecript 不存任务到数据库；前端持有 `task_id` 即可断网恢复，符合"≤ 100 元月成本"约束

#### 商业化不变量

引入第 7 干预点不破坏现有商业模型：
- v0.x 自托管：用户自填 ZHIPU_API_KEY，按官方价直扣（0.5 元/条），Seecript 不抽成
- v1.x hosted：可选"包月视频额度"作为加价模块（如 49 元含 80 条/月）
- v2.x 团队：团队额度池 + 配额管理

#### Step 3 · 爆款拆解与脚本生成

```
第 0 步【自动】  人设已由 Step 2 选定
   ↓
第 1 步【手动】  二选一 tab：上传视频 / 粘贴台词
   ↓
```

| 错误类型 | 用户看到什么 | 出路 |

## 五、使用场景 (Use Cases)

### 5.1 场景 A · 上班族转型短视频博主

> 小李，互联网公司产品经理，月入 1.2 万，想做"职场反内卷"博主。每天通勤 1 小时 + 周末 2 小时是全部可投入时间。

**用 Seecript 的一天**：

| 时间 | 动作 |
|---|---|
| 周六 9:00 | 在工作台点「生成新人设」，填"产品经理 / 反内卷 / 1 部 iPhone" |
| 9:05 | AI 出 3 个方案，选「反内卷打工人 · 用产品思维拆穿职场骗局」并采用 |
| 9:10 | 自动进入爆款拆解，人设已带入 |
| 9:12 | 上传一个收藏的对标视频 → 等抽轨 + 转写 |
| 9:14 | 拆解骨架出炉（Hook 痛点前置 + Body 3 段 + CTA 评论区互动） |
| 9:15 | 填 brief：60s · 紧凑 · 幽默；点开始 3 轮问答 |
| 9:16 | 答 3 题：① Hook 数字反差 ② Body 实拍演示 ③ CTA 打首字母 |
| 9:17 | 自动生成完整分镜脚本 |
| 9:18 | 点「下一步：标题车间」自动带入；生成 5 个标题 + 简介 + 标签矩阵 |
| 9:20 | 「采用」一个标题、复制简介、一键复制全部标签 → 去剪映 |
| 周一晚 | 视频发布 24 小时，评论 50 条 → 粘到评论分拣 → 8 条高价值 + 三种语气回复草案 |

**全程 30 分钟**完成原本需要 3 小时的工作流。

### 5.2 场景 B · 多账号矩阵运营

> 老王同时运营 3 个差异化账号（职场 / 美妆 / 育儿），每个账号都需要独立人设与脚本。

| 痛点 | Seecript 解法 |
|---|---|
| 容易把 3 个账号的语气写串 | 第 0 步**显式选人设**，问答与脚本生成都注入对应人设上下文 |
| 历史脚本难找 | 工作台「我的脚本项目」按时间倒序 + 标题预览 + 「展开 ↓」直接看完整脚本 |
| 想试试不同时长/节奏的版本 | brief 表单换组合参数重跑即可，3 分钟拿到对照版 |

## 六、产品价值 (Value Proposition)

### 6.1 核心价值主张

> **让一个人，干出一个 MCN 团队的产出质量。**

具体兑现为 4 个层次：

| 层次 | 用户感受 | 量化指标 |

## 八、商业化思考 (Commercialization)

### 8.1 目标市场规模

| 数据点 | 规模 | 来源 |
|---|---|---|
| **全网 KOC 活跃达人** | **突破 1000 万人** | 星子文化《2025 KOC 达人推广行业白皮书》 |
| **万粉以上创作者总量** | **1493 万**（增速放缓至 5%——精耕细作时代） | 克劳锐《2025 腰部达人商业化研究》 |
| **2025 年 KOC 投放预算** | **400 亿元，同比增长超 60%**（增长最快的投放分支之一） | 星子文化白皮书 |
| **KOC 达人主流月收入区间** | 3000-8000 元（年收入超 10 万人数 1 万人+） | 同上 |

**Seecript 的目标 SAM**：1000 万 KOC 中**月收入 3000-8000 元**的中长尾创作者，约**占 60%（即 600 万人）**——这群人最缺工具、最在意效率、对 MCN 服务买不起又用不上。

### 8.2 盈利模式（分阶段）

| 阶段 | 模式 | 价格锚 | 触发条件 |
|---|---|---|---|
| **v0.x（当前）** | **开源 + 自托管**——产品力验证期不收费，用极低门槛聚集种子用户与口碑 | 免费 | 当前 |
| **v1.x** | **hosted 单租户版** | 4G 实例约 50 元/月（覆盖算力 + 微利）| 单租户验证完成、月活 ≥ 500 |
| **v2.x** | **团队协作版**（多用户 / 云端历史 / prompt 微调）| 199 元/月 / 5 人席位 | 出现 5+ 个人客户主动询问"能否给团队用" |
| **v3.x** | **API + 集成生态**（接入剪映 / 万兴 / 第三方发布平台）按调用量计费 | 0.05 元/次 AI 调用 | DAU ≥ 200 |

**核心商业逻辑**：先用开源 + 自托管收割对**隐私敏感 + 低预算**的种子用户（约占目标群体 30%）；中期推出 hosted 收割对**易用性敏感 + 不愿运维**的主流用户（约占 50%）；后期通过 API 触达**愿意付费集成**的工具链上游（剪映 / B 端代运营公司）。

### 8.3 竞争优势

| 维度 | Seecript 优势 | 量化体现 |
|---|---|---|
| **产品形态** | 一条工作流串起 5 个环节，**对标爆款是切入口** | 同类产品要么单功能（仅做标题）要么发散对话（ChatGPT），**没有第二家把"拆爆款 → 出原创"作为主线产品** |
| **可控性** | 引导式问答硬限 ≤ 3 轮 | **100% 收敛**——通用 ChatGPT 用户平均要 7-12 轮才能拿到可用脚本 |
| **隐私优先** | 100% 浏览器本地存储 + 自托管开源 | 与 SaaS 创作工具相比，**用户的爆款台词、原创脚本不上传任何服务端数据库** |
| **轻量部署** | 后端单实例（4G 内存）跑全流程；视频抽轨在浏览器完成 | 单服务器月成本 ≤ 100 元，比同类 SaaS 服务端推理成本低**一个数量级** |
| **离线可演示** | mock 模式覆盖全部 6 个 AI 干预点 | 客户演示场地无外网也能跑通——**这是同类产品都做不到的** |

### 8.4 风险与对冲

| 风险 | 对冲策略 |
|---|---|
| 大模型 API 涨价 | LLMClient 抽象层支持任意 OpenAI 兼容协议；可一周内切换到 DeepSeek / Qwen / GLM / OpenAI 任一供应商 |
| 平台算法剧变 | 标题车间的 system prompt 与平台规则解耦，规则升级只需改 prompt 文件不动核心代码 |
| 头部 MCN 自研类似工具内部化 | 主打"开源 + 自托管"差异化——MCN 工具不开放，反而帮我们守住中长尾市场 |
| 用户对 AI 生成质量信任度下降 | 三层兜底 + 学习价值层（每个输出附方法论标签）让用户**逐步从依赖到独立**，而非锁死在工具里 |
