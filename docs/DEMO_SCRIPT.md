# Seecript · 演示视频介绍稿

> 受众：在读研究生本人面向评委 / 老师 / 同学口播
> 时长建议：15-18 分钟（可裁剪到 8-10 分钟精简版）
> 公网 demo：<https://seecript.zlhu.asia>
> 录制动线：完全按顶栏 6 个 Tab 从左到右走，每个页面内部按 UI 自上而下、从左到右展开

---

## 0 · 开场（30 秒）

各位老师好，我做的项目叫 **Seecript**——「**爆款结构迁移引擎**」。

一句话说清楚它做什么：**给一支爆款样例视频和一些零散素材，自动产出一条结构对齐、风格相近的新视频**。用户不需要懂剪辑，工具会把样例的段落骨架、节奏、情绪曲线"迁移"到用户自己的内容上。

---

## 1 · 我要解决什么问题（2 分钟）

短视频创作者的痛点是"看到爆款但学不会"——同样的题材，自己拍效果差很远。

差距通常不在画面好坏，而在**结构**：开场几秒钩住人、中段怎么递进、什么时刻给到情绪峰值、收尾如何转化。但"结构"过去无法量化，只能靠创作者拍多了才有手感。

我做的事情是把样例视频拆成**可计算、可迁移的结构特征**：

- **段落骨架**：起承转合 / 步骤式 / 盘点式 / 氛围推进 / 信息密集 / Vlog 日常 这 6 种叙事模式中选一种，再切成若干段，每段标 role（结构角色）+ theme（具体主题）
- **节奏曲线**：每个镜头的时长、切换密度、BGM 能量起伏
- **情绪曲线**：LLM 综合段落 / BGM / 镜头脚本 / 整片调性，给出 60 个采样点的强度曲线 + 峰值/低谷时刻
- **包装风格**：字幕样式、标题条、贴纸、转场、封面 这 5 个维度

用户给意图（brief）+ 自己的素材，系统就能用样例的"骨架"装上用户的"血肉"，输出一条新视频。

**核心信念**：**结构来自样例，内容服务用户**。

## 2 · 用户主链路（10 分钟，演示主体）

下面我按顶栏 6 个 Tab 从左到右走。每到一个页面，会按它内部的 UI 顺序逐块讲清楚：**这块在解决什么 / 用户在这里给什么（入参）/ 系统在这里给什么（出参）**。

顶栏 6 个 Tab 依次是：**首页 → 我的项目 → 素材与灵感 → 分析热门结构 → 创作工作台 → 我的创作偏好**。

### Tab 1 · 首页（intro.html · 30 秒带过）

打开站点先看到一张深色主题的产品落地页。视觉上 Hero 区是粒子飘浮 + 渐变发光的 Seecript 大字，副标语是一句话定位：

> **「上传你喜欢的视频，AI 自动拆解结构与节奏，换上你的素材一键导出同样精彩的作品」**

往下滚有 3 张价值卡：**看懂爆款 / 重组灵感 / 一键出片**——这是给陌生访客 30 秒理解产品的简化叙述。再往下是营销简化版的 4 步工作流图（选灵感 / 自动分析 / 拖拽剪辑 / 一键发布），它和我下面要讲的真实工作流不一一对应，是给非创作者看的"门面话"。

我点顶栏的「我的项目」进去，进入真实的产品。

### Tab 2 · 我的项目（HomePage · 1 分钟）

页面布局自上而下：**顶部操作条 → 项目网格**。

#### 顶部操作条

- 左侧：项目数量统计（"共 N 个项目"）+ 刷新中提示
- 右侧：「+ 新建项目」按钮

#### 项目网格（3 列）

每张卡片自上而下展示：**封面缩略区 + 状态徽标（草稿 / 方案已生成 / 成片已生成 三色）+ 项目名 + 参考样例 ID + 更新时间 + brief 摘要 + 三个动作按钮（打开 / 改名 / 删除）**。

**这一页在解决什么**：让用户对自己手上所有"在做的视频"一眼看清进度——哪些还在草稿、哪些方案出了、哪些已经渲染过。**用户每个项目下的素材、Plan、对话历史、AIGC 产物都按 project_id 隔离**，多项目并行做不混。

#### 新建项目对话框（点「+ 新建项目」唤起）

输入两个字段：**项目名称 + 视频用途简述**，再选一个**视频类型**（产品营销 / 知识科普 / 生活记录 等）。点确认 → 后端创建 project_id → 自动跳工作台。

**入参**：项目名 / 用途 / 视频类型。**出参**：一个空项目卡，状态 = 草稿。

---

### Tab 3 · 素材与灵感（LibraryPage · 2 分钟）

这个 Tab 是顶栏第二个核心枢纽——**所有"灵感"和"原料"都在这里**。页面顶部有一个 2 选 1 的子 Section 切换：**爆款样例 / 我的素材**。

#### 子 Section 1 · 爆款样例

布局自上而下：**来源 Tab（官方样例 / 我上传的）→ 视频类型筛选 chip（全部 / 营销 / 剪辑 / 动效）→ 卡片网格**。

**网格第一格永远是上传卡片**（虚线、点击或拖拽上传），后面是已有样例。

每张样例卡自上而下：**封面缩略图 + 视频类型 chip + 场景 chip + 标题 + 已拆解版本徽标（"已拆解 · N 个版本" / "还没分析"）+ 时长 + 镜头数 + 右下角播放按钮（弹出 PreviewModal 看原片）**。

**这一段在解决什么**：让用户决定"借鉴谁的结构"。
- **官方样例**：内置营销 / 剪辑 / 动效图三个领域已预拆好的样例，最快路径直接演到下一步
- **我上传的**：用户传一支看到的爆款，系统自动拆解，这是真正能体现"懂样例"的入口

点任意样例卡 → 跳到 `/decompose?sample=xxx`，进入下一个 Tab。

**入参**：样例文件 / 视频类型。**出参**：一条已就绪、可点开看的样例卡 + 后续可拆解的入口。

#### 子 Section 2 · 我的素材（AssetLibraryView）

布局自上而下：**Tab（我的素材 / 系统素材库）→ 操作条（上传 / 从系统素材库选 N 条 / 总数）→ 素材网格（4 列、可拖拽排序）**。

每张素材卡（MaterialCard）信息密度比样例卡高：

- 缩略图（视频取首帧 / 图片直显 / 音频用 🎵 占位）+ 左上角拖把手（可拖排序）
- 右上：**AIGC 来源 badge**（"AI 生图" / "AI 生视频"——区分用户自传 vs AI 产出）
- 右上：**media_type chip**（VIDEO / IMAGE / AUDIO）
- 右下：**highlight_score**（高光分 ≥0.75 黄色 ⭐ / ≥0.5 银色 / 隐藏）
- 左下：**preprocess_status**（⏳ 排队 / ⚙️ AI 切片中 / ⚠ 失败 / ✓ N 镜头）
- 底部：**recommended_section 浅色色带**（按推荐段位分色：opening / development / climax / closing）
- body：文件名 + × 删除 + 时长 + tags

**这是稿子里"理解素材"那段的真实落地点**：每张视频卡上的 "✓ N 镜头" 徽标，意思是这支视频已经被自动按内容切成了若干个分镜（最多 12 个），每个分镜独立写描述、估节奏强度、推荐去向；图片虽然没法切段，但同样跑一遍内容理解。底部那条彩色色带就是**系统给这条素材的去向推荐**——一眼能看出"这条素材在 plan 里最可能去哪段"。

#### SystemLibraryPicker 弹窗（点「从系统素材库选」）

弹窗里是一个共享素材池，用户能勾选最多 20 条克隆到自己项目里。**克隆生成新 material_id，与系统库独立**——之后用户对自己副本的修改不影响系统库，反之亦然。

**入参**：上传的视频/图/音频，或系统库的勾选。**出参**：一池"已被理解过的"素材，每条都有缩略图 / tags / 推荐段位 / highlight_score / 切片状态 ——后续在 plan 里能按内容自动分发到对应段落。

### Tab 4 · 分析热门结构（DecomposePage · 2 分钟）

从 Tab 3 点样例卡过来，或顶栏直接点「分析热门结构」都能进。这一页的目的是 —— **让用户和系统对齐"对样例的理解"**。理解错了，下游的迁移就一定错。

#### 顶部状态条

显示当前选中的样例 ID + 视频类型 + 当前 manifest 版本（v0 / v1）+ 「重新拆解」按钮 + 多版本切换 select。**每个样例最多保留 2 个版本**——方便用户对比"原始拆解 vs 我手动改过的版本"，超过会触发 slots_full 让用户主动选保留哪个。

#### SSE 流式拆解

用户没拆过 → 点「重新拆解」触发。前端通过 SSE 实时收到拆解进度（0-100%）+ ThinkingSteps（一条接一条 fade-in，让用户看到 LLM 在做什么）。完成后页面落到下面三个结果块。

#### 结果块 1 · 整片画像（AnalysisCard）

**问题**：这是什么类型的视频？

自上而下：**视频类型大字 + 结构模式名（如"钩子-放大-反转-收尾"）+ 拆解时间 + 版本徽标 + 段落色带可视化（4-5 段不同颜色 + 段名）+ 整片摘要 + 目标观众 / 调性 / 节奏 三 chip + 折叠"结构归纳"**。

**关键设计 · 理解优先**：让 LLM 先看完整片再切段，而不是按预设的"开头-中间-结尾"硬塞。所以艺术展宣传片就切成"展品揭幕→艺术家自述→行动呼吁"，不会被强行套成 hook/body/cta。

#### 结果块 2 · 段落+镜头表（DecomposeTable）

**问题**：它是怎么切段的？每个镜头在做什么？

表头：**# / 段名 / role chip / start–end / 时长 / 镜头数 / theme / summary / 关键 shots**。点击行 → 展开 shot 列表，每镜显示**缩略图 + 时间窗 + 字幕原文 + tags + subject + camera_technique**。行末挂 tempo chip（slow / medium / fast / peak / deceleration）。

**关键设计 · role + theme 双层**：role 是抽象骨架（任何视频都通用），theme 是具体语义（让用户一眼看懂）。

**关键设计 · 任意字段可编辑**：用户觉得 LLM 拆错了，当场改——改完落到当前 manifest 版本，不影响"原始拆解"那个版本，方便对比。

#### 结果块 3 · BGM 分析（BgmAnalysisCard）

**问题**：节奏和情绪怎么走的？

自上而下：**BGM 文件名 + 时长 + 是否检测到节拍 → bpm / key / energy_shape 三 chip → 高潮时间轴**（横条 + ⚡ 标每个 climax，按 kind 上色：climax/drop 紫红 / build_start 琥珀 / release/break 灰）**+ calm_segments 浅色块 + 结构注释**（intro 0-8s / verse 8-30s / chorus …）。

底部一句话提示：**「这条 BGM 的能量曲线会拷贝到你新视频的 BGM 上」**——这是节奏画像在 BGM 维度的直接体现。

**入参**：样例视频文件。**出参**：整片画像 + 段落镜头表 + BGM 节奏画像 + 情绪曲线（这个曲线在工作台里也会用到）。

### Tab 5 · 创作工作台（ComposePage · 4-5 分钟，主战场）

这是产品的核心工作台。顶部固定一个**三步进度条 WorkshopStepNav**：

> ① 选参考 + 写主题 → ② 生成内容轨 → ③ 多轨 + 出片

每个 step 顶部右侧挂 **VersionMenu**（保存 + 版本下拉，能保存命名快照、切回历史版本、删除）。页面右下角悬浮 **DraggableCommandFab**——「⌘K · 对话编辑小助手」按钮，可拖动定位、4px 阈值区分点击/拖动，是后面要重点讲的自然语言编辑入口。

---

#### 第 1 步 · 选参考 + 写主题（左输入 / 右输出 双栏）

##### 左栏 · 输入区（自上而下 5 件事）

**1. ReferencePicker（结构参考槽）**

横向并列两个固定槽 ——「样例 A」/「样例 B」。每个槽四种状态：空（拖拽提示 + 上传 / 从样例库挑 / 贴 URL）/ 上传中 / 拆解中 / 已就绪。已就绪槽显示缩略图 + 文件名 + 时长 + 分段数 + video_type chip + 「重新拆解」/「移除」。

**为什么是 1-2 个不是更多**：1 个槽迁移单一参考的结构；2 个槽允许"取这个的开场 + 那个的结尾"的混搭。再多就过载，模型也分不清主次。

**2. BriefInput（视频主题）**

textarea 200 字硬上限 + 字数计数器。占位提示："比如『介绍我的青铜器收藏，给文物爱好者看』"。

**3. VideoGoalInput（视频要求与目的）**

textarea 500 字硬上限。可选项——填了它，LLM 在改编结构时会按用户的目的去调样例骨架，而不是照搬。

**4. ClarifyPanel（多轮意图澄清，最多 3 轮）**

点「✨ 让 AI 先帮我想想」启动。SSE 流回吐 ThinkingSteps + 一组 question 卡（每张卡有 question 文字 + 单选/多选/自由文本输入 + 几个 suggestion chips 让用户一键选）。最多 3 轮，"第 N 轮 / 3 轮" 进度条 +「跳过本轮」/「继续追问」。

> 系统：这次的核心镜头要拍哪些具体物体？
>
> 用户：青铜鼎、玉器、明清画作

完成后给出 brief_stitched 文本预览，**红/绿 diff 显示哪几句来自用户、哪几句来自 AI 拼接**。用户点「采纳为最终 brief」。

最终 brief 长这样：「博物馆开馆 30 周年纪念片，**涉及（青铜鼎、玉器、明清画作）**」。括号里的物体清单会被抽成 **subject_anchors 白名单**——这是个**硬约束**：下游所有 plan 生成都必须把这几个物体安排进镜头，**漏一个都不行**。

**为什么要硬约束**：LLM 写文案爱用上位词、营销词、形容词替换具体物体（"展品" 替换 "青铜鼎"），用户最在意的镜头反而最容易被丢。subject_anchors 是反这个倾向的最后一道闸——**代码层兜底强插，不能仅靠 prompt"建议" LLM 去做**。

**5. ComposeSettingsPanel（折叠 details）**

设置创作约束：**目标时长 number（5-600 秒）/ 节奏 select（紧凑 / 平稳 / 慢叙）/ 字数密度 select（极简 / 默认 / 信息流密集）/ 调性 chips（温暖 / 锐利 / 戏谑 / 严肃 / 治愈 / 反转）/ 重点 chips（钩子前置 / 高潮居中 / 反转结尾 / 卖点平铺）**。

里面嵌一个 **FrameDesignPicker**：6 种预设画风 chip（自由配色 / 高能社交 / 治愈生活 / 商务科普 / 海报大字 / 电影胶片）+ 展开细调（6 色 palette、动效密度、颗粒/胶片/暗角 toggle、备注）。

**底部「✨ 生成方案」大蓝按钮**，调用 `/plan/build`，10-30 秒返回。

##### 右栏 · 输出区（生成完后看到）

**1. StructureMapPanel（结构对照图）**

ReactFlow 栅格图，节点不可拖、不可缩放。**单样例**：左 = 样例 / 右 = 新方案；**双样例**：左 = 样例 1 / 中 = 新方案 / 右 = 样例 2。每个节点是一段 ：sideLabel·段名 / 关系 chip（命中 / 新增段落 / 未沿用）/ theme / 时长 + 镜头数。**同段同色边连两侧**——用户能一眼看到"我的新方案是怎么从样例脱胎而来的、哪些段落是新增的、哪些样例段落被丢弃了"。

**2. AdaptedSectionList（每段汇总卡）**

每段一张：role 色点 + section_short + theme + tempo chip + **段汇总状态 chip**（✅ 完整 / ⚠️ 还差点 / ❌ 缺素材 / — 不需要补）+ **三色质量条**（精准 emerald / 待修补 amber / 缺匹配 slate，按 shots 的 match_quality 比例画）+ content_description + adaptation_note（"改编思路：..."）。再下面是这段的 sectionGaps 列表，每个 gap 描述需要补什么、当前是否已配上素材。

**3. EmotionCurveCard（情绪曲线卡）**

可折叠。展开态是 320×120 LineChart：**60 点情绪强度曲线 + 段色带底层 + 红色高潮 ReferenceDot + 灰色低谷 ReferenceDot + 红色播放头**。**注意**：拆解时算的是"原片的情绪"，这里算的是"用户意图 + 迁移倾向作用后的目标情绪"——如果用户在前面选了"情绪增强 +20-30%"，新片的情绪曲线整体应该比原片抬高。

##### 这一步在做什么 · 三层混合 Plan 生成的核心

`/plan/build` 内部走三层：

**第一层 · 硬约束（代码确定性保底）**：段落总时长归一（所有段时长之和 = 用户设定目标，多了少了代码强行重算比例）/ 段落数 + 角色名合法（按 structural_pattern 卡白名单）/ subject_anchors 强插（用户白名单里的物体必须全部出现 ≥1 次，缺哪个就在最匹配的段里硬塞进 ShotPlan.subject）。

**第二层 · Prompt 工程（喂给 LLM 9 类信号）**：

| 信号 | 作用 |
|---|---|
| 样例画像（archetype / 叙事概要 / tone） | 告诉 LLM 这是什么风格的片 |
| 段落骨架 | 当结构模板用 |
| 镜头目标 kind 频次摘要 | 知道原片有多少 graphic / object / text 镜头，但不传具体 name 避免抄内容 |
| 用户 brief + video_goal | 知道用户要什么 |
| subject_anchors 白名单 | 知道必须拍的物体 |
| 创作设置 | 时长 / 平台 / 调性 / CTA / 关键词 |
| 迁移倾向 hint | mirror（平淡复刻）/ amp_emotion（情绪+20-30%）/ amp_pace（节奏-15%） |
| 样例分析 highlights + improvements | 让 LLM 保留样例强点、规避弱点 |
| **原片节奏画像** | 段时长比例 + 节奏密度 + 情绪曲线峰值时刻 |
| 个性 KB 注入 | 用户过往项目蒸馏出的偏好（详见 Tab 6） |

**第三层 · 节奏画像驱动的结构对齐（关键设计）**：

注意上面表格里加粗的"原片节奏画像"——这是**结构迁移最核心的传递通道**。

举例：原片在 18 秒处情绪到顶（BGM 鼓点 + 关键句叠加 + 快速切镜），系统把这个峰值位置一并喂给 LLM，要求新片在按时长比例换算后的对应位置（30 秒片就是 ~9 秒处）也安排一个能撑起情绪峰值的段落 —— 而不是让 LLM 自己瞎猜结构。

为什么不再用"参考风格图"：**结构迁移是结构的事，画面归画面**。把一张静态参考图喂给写结构的 LLM，它顶多在文案里多几个形容词，对段落骨架没有真正的影响。换成节奏画像后，LLM 拿到的是"这片每一段的能量怎么走"——这才是结构能用上的信号。**不抄画面，只抄节奏**。

确认右栏方案 OK → 点底部「下一步：进入工作台」，跳第 2 步。

#### 第 2 步 · 生成内容轨（FourTrackBoard sections 模式 · 重点讲）

承接第 1 步：plan 一生成完，**系统就把素材库里的视频分镜和图片按 caption 与段落的语义匹配度做了一遍全局最优分配**——同一支视频的不同分镜可以分别去到不同段落（开场推门去 opening / 中段对话去 development / 结尾拉远去 closing），而不是整支视频被绑定到一个位置。用户进第 2 步看到的内容轨**不是空的**，已经按内容配好了第一版。

##### 左栏 · 时间线（FourTrackBoard sections 模式）

自上而下：**时间标尺 → 参考样例轨（折叠展示节奏对照） → 内容轨 sections**。

内容轨上每段一块大色块（8 色循环），块内信息：**section_short + theme + 时长 + ✏ 段编辑按钮 + ▾ 展开按钮 + 多镜数量 badge（"3 镜"）**。

展开后每镜显示一个小卡：**subject + 时长 + 质量圆点（绿 ≥0.30 / 橙 0.10-0.30 / 灰 <0.10）+ fit_score chip（≥0.7 emerald / ≥0.4 amber / <0.4 rose）+ 段落转场 badge**（hard_cut / dissolve / slide / zoom / whip / wipe）。

**这就是"匹配度可视化"的真正位置**——绿/橙/灰圆点让用户一眼看出哪段配得准、哪段还需修补。底部 legend：● 已补 / ● 待调 / ● 缺失。

##### 右栏 · 编辑面板（按用户点击的位置切换）

- 点段块 ✏ → **SectionEditDialog**：改段主题（≤80）+ 段描述（≤400）。提示用户"段时长改回第 1 步重生 plan"。
- 点小镜 → **ShotEditDialog**：改 subject / visual / narration / camera_technique（每个字段独立上限）+ 4 tab 换源（用户素材 / AI 单图 / AI 视频 / 字卡）。
- 点 4 个换源按钮 → **SwapSourceDialog**：精简版换源弹窗，4 tab 同上，用户素材模式下显示缩略图网格 + 多镜素材的 shot picker（再选哪一镜）。
- 点 gap → **GapPreviewDialog**：看样例对应位置的截图大图，理解"这一段应该长什么样"。

##### 三路缺口补全（第 2 步另一重点 · 重点讲）

橙点 + 灰点的段落 = "缺口"。每个缺口用户能选三种补全方式：

###### 5A · 换素材 · FillRerankPanel

**思路：系统先把脏活干完，用户只做"是否同意"的决策**。点开候选列表，**RerankStrategyPicker** 按 fit_score 重排（段位推荐 + 时长贴合 + 内容描述命中 + 高光分综合排序）。点哪条就换成哪条——**不让 AI 替用户做主，但把"挑哪条"的认知负担降到最低**。

弹出的换素材预览：双 TimelineTrack 对照（原方案 / 换后）+「取消」/「采纳这次替换」。

###### 5B · 字卡画面 · FillCopyPanel

用户某段素材实在没有但又必须有内容时，用一张设计感字卡填空。

UI 进展：**idle**（输入风格 hint，如"暗黑科技风"，或跳过用默认）→ **analyzing-outline**（ThinkingSteps）→ **outline**（左表单：主标题 ≤24 / 副标题 ≤40 / 4 字体（粗黑/宋体/手写/科技等宽）/ 4 版式（居中/顶部/底部/分栏）/ 4 背景模式（纯色/渐变/模糊图/暗罩）/ 三色 picker（bg/text/accent）/ 4 动画（淡入/打字机/抖动/弹出）/ emoji_decor ≤3 / duration slider 1.5-15s。右 CardPreview 实时预览）→ **generating** → **result**（"AI 已把字卡烧制成视频片段，2-5 秒"）。

###### 5C · AI 视频生成 · FillAigcPanel（最有意思的一路）

PhasePill 三阶段：**1.参考图 → 2.提示词 → 3.生成**。

具体链路：

1. **生成提示词**：LLM 看段落上下文 + brief + subject_anchors 写出英文 t2v prompt
2. **生成参考图**（可选）：用 Seedream 出一张图当 first_frame 风格基准
3. **抽前段尾帧**（可选）：抽 main_track 上一个 Scene 的最后一帧，作为本段 first_frame——**让前后两段视频画面无缝衔接**，不会有突兀的画风跳变
4. **Seedance 生视频**：豆包 T2V 单段最长 6.5 秒。如果本段需要 12s / 18s，系统拆 chunk，**每段尾帧自动作为下段首帧串成长视频**

**这是产品最有技术含量的功能**——豆包 T2V 单调用只能出 6.5s，要做出连贯长视频必须自己设计**首尾帧链**。我把它做成对用户透明：用户只看到时长选项，背后是 1/2/3 个 chunks 自动串接。

UI 进度：**IdleStage** →「开始分析 ✨」→ **SpecStage**（每个 spec 卡：caption + ratio + 提示词 textarea ≤300 + 3 个按钮"上传 / 素材库选图 / AI 出图"）→ **PromptStage**（缩略图条 + 提示词 textarea + 720p 提示 + image 模式分镜清单 + **尾帧承接 checkbox** 仅 video 模式且上一镜已就绪）→ **FillStatusCard**（实时状态 + chunks_count + 任务号 + 8s × 30 自动轮询 → 预览就绪 → ✓ 已应用 / ↻ 重新应用）。

###### 5D · 衍生：AI 生图再渲染（带运镜）

如果用户已经有一张满意的静态图（或刚走完单出图路径），可以选「**AI 生图再渲染**」把图变成有运镜的视频段——避免"PPT 静帧"。

**运镜由 LLM 为这一镜头单独决策**：综合本镜主体、本镜在段落里的角色（开场 / 推进 / 高潮 / 收尾）、整片节奏 tempo，输出推近 / 拉远 / 上摇 / 下摇 / 横摇左 / 横摇右 / 视差等运镜方式 + 强度等级。**不是用一套粗规则套所有镜头**——开场的"展品揭幕"和高潮的"人物特写"该用什么运镜完全不同，由 LLM 看完镜头本身的内容来决定。**支持的 6 个方向 + 视差在两条渲染路径上都跑得通，不会因为后台走了哪条渲染路径就退化成静帧**。

##### ⌘K 自然语言编辑（第 2 步作用域）

按 ⌘K 唤出 ComposeCommandBar 全屏对话框。**第 2 步作用域**：段文案 / 段时长 / 删段 / 重排 / 分镜画面 / 口播 / 字卡 / 包装 / 转场 / BGM / 重排素材——内容轨全开，但禁 AIGC 重生（避免误烧钱）。

> "把第 2 段缩短一半，加点紧迫感"
>
> "结尾加一个转场到产品特写"

LLM 输出**操作 ops**（修改字段、增删 Scene、换素材等），但不直接落地——前端先 **dry-run 预览**给用户看 diff 卡，用户点「应用 N 项」才提交。每次 apply 生成新 plan_id 入撤销栈，撤回随时可回到任何历史版本。被 dismissed 的 ops 也喂给后端做负样本。

##### 第 2 步底部出口

底部「进入第 3 步」按钮——**只有内容轨完全补完才解锁**，否则上方进度条会标"内容轨还有 N 段未补完"提示。

#### 第 3 步 · 多轨 + 出片（FourTrackBoard shots 模式）

##### 左栏 · 4 轨时间线（shots 模式）

时间标尺 + 红色播放头（来自右栏 PlanPlayer 的 onTimeUpdate）。

**1. 内容轨**（每个 Scene 一块色块）：thumbnail 优先级 = text_card_spec > AIGC fill cover > AIGC fill image > scene aigc image > material thumbnail > source 字符；色块上叠 transition badge。

**2. 字幕轨**（subtitle_enabled toggle）：每 scene 一段字幕色块，字卡画面段显示"无需字幕"。点击 → SubtitleEditPopover：textarea + 字数提醒（duration × 4 建议字数）+ 超长 amber 警告。

**3. 口播轨**（voiceover_enabled toggle）：**5 voice 选项**（清新 / 温柔 / 小雨 / 解说 / 学忆）+ 「一键全段合成」+ 每 scene 状态色（ready emerald / pending amber / empty）+ 角标（🔊 试听 / ↻ 重合成 / × 删除）。

**4. 包装轨**（assignPackagingLanes 3 lane 贪心排布）：**5 种 PackagingItem kind**——subtitle / title_bar / sticker / transition / cover。每个 PackagingItemBlock 有左/右 resize 把手 + 中间拖动 + 点击编辑（PackagingItemEditDialog）+ × 删除。「✨ 添加组件」按钮 → PackagingSmartAddButton（选 target scene + kind）。

**5. BGM 轨**：顶条 = BGM 名称 +「上传/选择」（→ BgmPickerDialog）+「换曲」+「清除」+ 音量 slider。BgmStripe = calm_segments 浅色背景 + climaxes ⚡（按 kind 上色）+ 形状渐变（flat / single_peak / build_up / multi_peak / wave）。

##### 右栏 · 输出 + 控制

**1. PlanPlayer**（Remotion 1080×1920 9:16 30fps）：实时预览整片，onTimeUpdate 喂左栏播放头。

**2. PackagingPanel V2 · 5 维度多候选**（**重点修正原稿事实**：包装层不是"一次给 6 种风格全套方案"，而是 **5 个独立维度 × 多候选**）：

- 顶部 4 个 **Preset 卡**：minimalist 极简 / energetic 活力 / info_feed 信息流 / dialogue 对话——每张含 tagline，点击立即触发后端跑 5 维度推荐。
- 折叠 CustomPanel：6 transition 风格 toggle / max_transition_duration slider / 字幕字号 / 字幕位置 / 字幕背景 / llm_temperature。
- 5 个 DimSection（按维度逐个挑）：
  1. **字幕样式**（single 选）：大/中/小 + 顶/中/底 + 无底/阴影/渐变 + 双语 + rationale
  2. **标题条**（multi 选）：彩色 preview + section 短标 + 起止时间 + position
  3. **贴纸**（multi 选）：同上
  4. **段落转场**（per-bundle single）：起止 + 段落 pill + ALL_TRANSITIONS（6 风格 + duration）+「+ 选风格」CatalogPicker
  5. **封面**（single 选）：rendered preview + 4 layout（center/left/stacked/split）+ title + subtitle + palette swatches

每个维度独立选 → 底部「应用所选」一次性应用回时间线。

**3. TransitionStylePicker**（点包装轨 ⇆ 节点）：6 转场风格卡 + duration slider。

**4. PackagingItemEditDialog**：text + 起止时间 + 颜色 picker + 7 种 position 位置。

**5. BgmPickerDialog**：上传新曲（≤20MB）+ 资产列表（标题 + 时长 + peak + 就绪/分析中/失败 chip + 选中走 PATCH /plan/{id}/bgm）。

**6. CatalogPicker**（HyperFrames 风格库）：80vh 弹窗，category tab + tag filter + 网格 hover 自动播 preview_video。

##### ⌘K 第 3 步作用域

**禁动内容轨**，但字幕 / 转场 / BGM / 包装 / 重排素材 / 重出字卡 / 重生 AI 生图 全开。**为什么作用域要分**：第 2 步用户在调结构，第 3 步用户在精修——精修阶段误改大结构是最痛的事故，作用域硬切就是为了防这个。

##### 渲染

底部「↓ 渲染并下载」蓝大按钮 + 副信息「渲染需要 ~30-90 秒，期间可继续编辑别的」。点击触发 RenderProgress 6 步：

| 步骤 | 进度 | 做什么 |
|---|---|---|
| 1. 准备 (`prepare`) | 8% | 解析 Plan、下载远端 AIGC 视频到本地、生成字卡图 |
| 2. 主轨拼接 (`ffmpeg_concat`) | 28% | 各 Scene 按段裁剪 + 归一画幅 + 拼成主视频 |
| 3. 主轨直通 (`seedance_extend`) | 48% | 等待豆包 Seedance 生成完成、串接首尾帧 chunks |
| 4. 包装渲染 (`remotion_render`) | 70% | Remotion 渲染**透明 WebM**（字幕 / 标题条 / 贴纸 / 转场 / 封面） |
| 5. 叠加输出 (`ffmpeg_overlay`) | 88% | 透明 WebM overlay 到主轨上 |
| 6. 收尾 (`finalize`) | 99% | 输出最终 mp4 + 自动生成封面图 |

每步有动画 + 实时进度 + 完成后绿勾。**关键设计**：包装层走 Remotion 渲染**透明 WebM 而不是直接画在主轨**——这样字幕样式可以独立改、独立换、独立调透明度，不影响主视频。

最终产物：一支 30 秒 9:16 的 mp4 + 一张封面图 + 「保存到本机」/「再来一版」按钮。

### Tab 6 · 我的创作偏好（KnowledgePage · 1-2 分钟）

这一页是**软性差异化的兑现位置**——把"用得越多越懂你"做成用户能看见、能开关的页面。三块内容自上而下：

#### Section 1 · 用户级设置

单 checkbox：**「实时蒸馏」（realtime_distill_enabled）**——每次出片完成时，AI 自动从我的成片回流总结我的创作偏好，写入对应项目的偏好库。下次相似项目自动注入。**关掉它就只写 trace 不出 KB**——给用户一个总开关。

#### Section 2 · 默认偏好（始终生效）

「内置 · 不可关」灰 chip + default_kb_description 文本块——告诉用户"系统内置了哪些通用偏好（比如开场不超过 3 秒、CTA 收尾这类通用规则）"，让用户对"为什么我什么都没设也有偏好"有解释。

#### Section 3 · 项目偏好

计数器：「N 个项目 · 当前注入 M 条偏好」。

每个项目一行折叠卡：

- **checkbox**（is_top10 = 最近 10 个完成项目时 disabled，永久启用）
- **状态 chip**：「自动启用（最近项目）」/「已启用」/「未启用」
- **项目标题 + rules 数 + video_type + summary + 最近渲染时间**
- **「查看规则 ▾」** → 展开 rules 列表，每条一行：**SCOPE_LABEL chip**（structure 段落结构 / source 镜头来源 / narration 口播文案 / pacing 节奏与时长）+ # 编号 + 规则文本

下次新项目生成 plan 时，**这里勾选启用的项目偏好会自动注入 prompt 顶部**——LLM 沿用我过往验证过的偏好，用得越多越懂我。

撤回的编辑（用户在 ⌘K dry-run 预览后说"不要这个方向"）也会被采集为**负信号**："避免 X"，避免下次再走错路。

## 3 · 功能创新点回顾（2 分钟）

不讲技术栈，只讲**用户能感知到的功能差异化**：

1. **理解优先的样例拆解**：先让 AI 看完整片再切段，6 种叙事模式 + 17 个角色名，对真实视频的适配度远高于"开头-中间-结尾"硬模板。在 **Tab 4 分析热门结构**里能直接看到这个产物
2. **可手动修正的拆解结果**：拆错了，用户当场改，系统记住改动，新版本立即生效；多版本 manifest 让"改之前 vs 改之后"可对比
3. **subject_anchors 硬约束**：用户白名单里的物体一个都不会被 LLM "营销词替换"丢掉。**Tab 5 工作台第 1 步 ClarifyPanel** 是它的入口
4. **素材按内容自动分发 + 分镜匹配度可视**：上传的视频在 **Tab 3 我的素材**里就被自动按内容切成多个分镜、写描述、推荐段位；Plan 完成的瞬间已被自动配上最匹配的分镜，**同一支视频的不同分镜可以分别去到不同段落**；**Tab 5 工作台第 2 步**每个 Scene 卡显示绿/橙/灰匹配度圆点 + 三色质量条，让用户对"哪段配得准"有直观反馈
5. **节奏画像驱动的结构迁移**：把样例的段落时长比例、节奏密度、情绪峰值时刻直接喂给 LLM，让它在新片对应位置安排能撑起情绪的段落 —— **不抄画面，只抄节奏**，这是结构能真正用上的信号
6. **三路缺口补全**：人工挑 / 字卡 / AIGC 同一交互入口，用户按需选；FillRerankPanel / FillCopyPanel / FillAigcPanel 三个 panel 在第 2 步并排可见
7. **AIGC 首尾帧链 + 每镜头 LLM 直选运镜**：解决豆包 T2V 单段 6.5s 限制，自动串成长视频画面无缝衔接；"AI 生图再渲染"路径上，每个镜头的运镜（推/拉/平移/视差）由 LLM 综合本镜主体 + 角色 + 整片节奏单独打分，两条渲染路径都跑通同一套运镜，不会退化成 PPT 静帧
8. **LLM anchor + 规则插值的情绪曲线**：把"情绪"这个看不见摸不着的东西，变成 60 点曲线 + 标好峰值位置的可视化产物。**Tab 4 / Tab 5** 都能看到曲线
9. **dry-run + 撤销栈的 NL 编辑**：⌘K 用户随便说，先看效果再决定要不要——不会有"AI 自作主张改坏了"的焦虑；第 2 步 / 第 3 步作用域硬切防误改
10. **5 维度独立可挑的智能包装**：包装层不是一锅端的 6 套全包风格，而是字幕 / 标题条 / 贴纸 / 转场 / 封面 5 个维度独立多候选；4 种 Preset 给"懒人模式"，5 个 DimSection 给"细调模式"
11. **跨项目知识沉淀**：用得越多越懂你，**Tab 6 我的创作偏好**把这件事做成了用户能开关、能看规则全文的页面，不是黑盒后台

## 4 · 收尾（30 秒）

整个项目的核心思想是把**"看爆款学不会"**这件事的瓶颈从**人的手感**转移到**系统的结构理解能力**——只要 AI 能把样例拆得足够准，把用户意图理解得足够具体，把硬约束写得足够死，就能把"结构迁移"这件事做成一个普通用户能 5 分钟上手的工具。
