# Seecript · 演示视频介绍稿

> 受众：在读研究生本人面向评委 / 老师 / 同学口播
> 时长建议：15-18 分钟（可裁剪到 8-10 分钟精简版）
> 公网 demo：<https://seecript.zlhu.asia>

---

## 0 · 开场（30 秒）

各位老师好，我做的项目叫 **Seecript**——「**爆款结构迁移引擎**」。

一句话说清楚它做什么：**给一支爆款样例视频和一些零散素材，自动产出一条结构对齐、风格相近的新视频**。用户不需要懂剪辑，工具会把样例的段落骨架、节奏、情绪曲线"迁移"到用户自己的内容上。

---

## 1 · 我要解决什么问题（2 分钟）

短视频创作者的痛点是"看到爆款但学不会"——同样的题材，自己拍效果差很远。

差距通常不在画面好坏，而在**结构**：开场几秒钩住人、中段怎么递进、什么时刻给到情绪峰值、收尾如何转化。但"结构"过去无法量化，只能靠创作者拍多了才有手感。

我做的事情是把样例视频拆成**可计算、可迁移的结构特征**：

- **段落骨架**：起承转合 / 步骤式 / 盘点式 / 氛围推进 / 信息密集 / Vlog 日常 这 6 种叙事模式中选一种，再切成若干段，每段标 role（结构角色）+ theme（具体主题）
- **节奏曲线**：每个镜头的时长、切换密度、BGM 能量起伏
- **情绪曲线**：LLM 综合段落 / BGM / 镜头脚本 / 整片调性，给出 60 个采样点的强度曲线 + 峰值/低谷时刻
- **包装风格**：字幕样式、转场偏好、封面调性

用户给意图（brief）+ 自己的素材，系统就能用样例的"骨架"装上用户的"血肉"，输出一条新视频。

**核心信念**：**结构来自样例，内容服务用户**。

---

## 2 · 用户主链路（10 分钟，演示主体）

我按用户操作顺序走一遍，每一步重点说清楚**这个功能在解决什么 / 用户给什么 / 系统给什么**。

### Step 0 · 创建项目

用户在首页点"新建项目"，输入项目名。

之后所有的素材、Plan、对话历史、AIGC 产物都按这个 project 隔离，互不污染。多项目并行做不混。

---

### Step 1 · 选样例视频

两条路径任选一条。

**路径 A · 用内置样例库**：营销 / 剪辑 / Motion Graph 三个领域各有一条预拆解好的样例，立即可用，最快路径直接演到下一步。

**路径 B · 自己上传**：用户传一支看到的爆款，系统自动拆解。这是真正能体现"懂样例"的入口。

#### 拆解功能设计

拆解不是简单切一下镜头。它要回答 4 个问题：

| 问题 | 系统给出的答案 |
|---|---|
| 这是什么类型的视频？ | 整片 archetype + 叙事概要 + 调性 + structural_pattern（6 选 1） |
| 它是怎么切段的？ | role + theme 双层段落（如：opening / "展品揭幕"，development / "艺术家自述"） |
| 每个镜头在做什么？ | tags + subject（拍的什么） + camera_technique（运镜） + targets（画面里的关键物体） |
| 节奏和情绪怎么走的？ | 每镜时长曲线 + BGM 能量 + 60 点情绪曲线 + 高潮位置 |

设计上的关键取舍：

- **理解优先**：让 LLM 先看完整片再切段，而不是按预设的"开头-中间-结尾"硬塞。所以艺术展宣传片就切成"展品揭幕→艺术家自述→行动呼吁"，不会被强行套成 hook/body/cta。
- **role + theme 双层**：role 是抽象骨架（任何视频都通用），theme 是具体语义（让用户一眼看懂）。
- **多版本 manifest**：每个样例最多保留 2 个版本，方便用户对比"原始拆解 vs 我手动修过的版本"。

---

### Step 2 · 在拆解结果页查看 + 修正

用户跳转到 `Decompose` 页，看到：

- **节奏曲线图**（紫色综合情绪线 + 灰色 BGM 能量虚线 + 红心标峰值 + 灰圆标低谷）
- **段落卡片**（每段：role + theme + 时长 + 包含哪些镜头）
- **每个镜头**：缩略图 + tags + 字幕原文 + 主体 + 运镜手法
- **整片画像**：archetype / 叙事概要 / tone

任何字段不准都可以**直接编辑**——这一步的目的不是只展示结果，而是让用户和系统**对齐对样例的理解**。理解错了，后续迁移就一定错。

---

### Step 3 · 进入工作坊 · 输入意图

用户进 `Compose` 工作坊（核心工作台，分 4 步导航：意图 → Plan → 编辑 → 渲染）。

#### 3A · 意图澄清（ClarifyPanel）

用户初始输入往往很模糊（"我想拍博物馆开馆"），LLM 多轮提问帮用户**把模糊变具体**：

> 系统：这次的核心镜头要拍哪些具体物体？
>
> 用户：青铜鼎、玉器、明清画作

最终把 brief 写成：「博物馆开馆 30 周年纪念片，**涉及（青铜鼎、玉器、明清画作）**」。

括号里的物体清单会被抽成 **subject_anchors 白名单**——这是个**硬约束**：下游所有 Plan 生成都必须把这几个物体安排进镜头，**漏一个都不行**。代码层会兜底强插，不能仅靠 prompt"建议"LLM 去做。

**为什么要硬约束？** LLM 写文案爱用上位词、营销词、形容词替换具体物体（"展品" 替换 "青铜鼎"），用户最在意的镜头反而最容易被丢。subject_anchors 是反这个倾向的最后一道闸。

#### 3B · 上传素材

用户传自己的视频/图片/音频，系统不是简单存档，而是**真正"看懂"素材**：

1. **视频按内容切段**：识别画面变化把一支视频切成若干分镜（最多 12 个），每个分镜独立处理 —— 同一支视频里"开场推门 / 中段对话 / 结尾拉远"这三种内容会被分开，**可以分别去到 Plan 的不同段落**，而不是整支视频被绑定到一个位置
2. **每个分镜写描述**：LLM 看每段画面写一句中文描述（"白衣女主推开木门走入厅堂"）+ 估算节奏强度
3. **每段推荐去向**：综合画面内容与节奏，推荐这段最适合放在 opening / development / climax / closing 哪个段位
4. **图片同样走理解**：图片虽然没法切段，但同样跑一遍内容理解 —— 写一句描述 + 推荐段位 + highlight_score，**与视频分镜在同一池子里参与匹配**

**关键差异**：素材不再是"一坨待挑选的池子"。Plan 生成完成的瞬间，**视频的不同分镜与图片已经被自动按内容匹配到对应段落**——用户进 Compose 看到的内容轨不是空的，而是已经按内容配好了第一版。

---

### Step 4 · 生成 Plan（结构迁移核心 · 重点讲）

这一步是整个产品的核心。

**用户操作**：选好样例 + 写好意图 + 设置目标参数（30 秒 / 抖音 / 9:16 / 紧凑+情绪增强 / CTA = "周末来看"），点"生成 Plan"。

**功能设计 · 三层混合**：

#### 第一层：硬约束（代码确定性保底）

不管 LLM 输出什么，这些都不让步：

- **段落总时长归一**：所有段时长之和必须等于用户设定的目标总时长（如 30 秒），多了少了代码层强行重算比例
- **段落数 + 角色名合法**：按 structural_pattern 卡白名单（戏剧四段式只准用 4 个角色，盘点式才允许 item_1 / item_2…）
- **subject_anchors 强插**：用户白名单里的物体必须全部出现 ≥1 次，缺哪个就在最匹配的段里硬塞进 ShotPlan.subject

#### 第二层：Prompt 工程（让 LLM 知道全部上下文）

喂给 LLM 的不是只有"样例段落"，而是 9 类信号：

| 信号 | 作用 |
|---|---|
| 样例画像（archetype / 叙事概要 / tone） | 告诉 LLM 这是什么风格的片 |
| 段落骨架 | 当结构模板用 |
| 镜头目标 kind 频次摘要 | 知道原片有多少 graphic / object / text 镜头，但不传具体 name 避免抄内容 |
| 用户 brief + video_goal | 知道用户要什么 |
| subject_anchors 白名单 | 知道必须拍的物体 |
| 创作设置 | 时长 / 平台 / 调性 / CTA / 关键词 |
| 迁移倾向 hint | mirror（平淡复刻）/ amp_emotion（情绪+20-30%）/ amp_pace（节奏-15%）|
| 样例分析 highlights + improvements | 让 LLM 保留样例强点、规避弱点 |
| **原片节奏画像** | 段时长比例 + 节奏密度 + 情绪曲线峰值时刻 —— 让 LLM 知道这片"哪几秒是高潮"，迁移时按比例换算到新片对应位置，**不抄画面，只抄节奏** |
| 个性 KB 注入 | 用户过往项目蒸馏出的偏好（详见 Step 8） |

#### 第三层：节奏画像驱动的结构对齐（关键设计）

注意上面表格里加粗的"原片节奏画像"——这是**结构迁移最核心的传递通道**。

举个例子：原片在 18 秒处情绪到顶（BGM 鼓点 + 关键句叠加 + 快速切镜），系统会把这个峰值位置一并喂给 LLM，要求新片在按时长比例换算后的对应位置（30 秒片就是 ~9 秒处）也安排一个能撑起情绪峰值的段落 —— 而不是让 LLM 自己瞎猜结构。

为什么不再用"参考风格图"？因为**结构迁移是结构的事，画面归画面**。把一张静态参考图喂给写结构的 LLM，它顶多在文案里多几个形容词，对段落骨架没有真正的影响。换成节奏画像后，LLM 拿到的是"这片每一段的能量怎么走"——这才是结构能用上的信号。

**输出 Plan**：每段标了 role + theme + 时长 + summary + 一组 ShotPlan（每个 shot 的预期 subject / kind / 时长），main_track 是按段顺序串起来的 Scene 列表。这时候很多 Scene 还是"空缺"——没素材、需要补。

#### 同时也算情绪曲线

Plan 生成完会再跑一次情绪打分。**拆解时算的是"原片的情绪"，Plan 阶段算的是"用户意图 + 迁移倾向作用后的目标情绪"**。如果用户选 amp_emotion，新片的情绪曲线整体应该比原片抬高 20-30%。曲线在工作坊页面展示，用户能直观看到峰值位置和强度。

### Step 5 · 缺口补全（三路并行 · 重点讲）

承接 Step 3B 的"素材自动按内容分发"——Plan 一生成完，系统就把视频分镜和图片**按 caption 与段落的语义匹配度**做了一遍全局最优分配。每个 Scene（镜头）卡片右上角有一个**匹配度圆点**，让用户一眼看出哪段配得准、哪段还需修补：

- 🟢 **绿点 (good)**：内容描述与段落主题高度吻合，可直接用
- 🟠 **橙点 (weak)**：勉强匹配，建议人工换一条
- ⚪ **灰点 (missing)**：完全没匹上，已用字卡占位等待补全

橙点 + 灰点的段落就是"缺口"。每个缺口用户能选三种补全方式（核心创新点之一）：

#### 5A · 换素材（系统已自动配 + 用户微调）

**思路：系统先把脏活干完，用户只做"是否同意"的决策**。

主轨上每个 Scene 已经被自动配好最匹配的素材分镜（视频/图片都参与匹配）。如果某段的圆点是橙或灰，用户点开候选列表，系统按 fit_score 重新排好序（段位推荐 + 时长贴合 + 内容描述命中 + 高光分综合排序）——用户点哪条就换成哪条。**不让 AI 替用户做主，但把"挑哪条"的认知负担降到最低**。

#### 5B · 字卡画面

LLM 出 3 个候选文案（标题 + 副标题 + 配色建议），用户选一个，系统渲染成图片当镜头素材。**用途**：用户某段素材实在没有但又必须有内容时，用一张设计感字卡填空。

#### 5C · AIGC 生成（豆包 Seedance T2V / Seedream T2I）

最有意思的一路。**4 步链路**：

1. **生成提示词**：LLM 看段落上下文 + brief + subject_anchors 写出英文 t2v prompt
2. **生成参考图**（可选）：用 Seedream 出一张图当 first_frame 风格基准
3. **抽前段尾帧**（可选）：抽 main_track 上一个 Scene 的最后一帧，作为本段 first_frame——**让前后两段视频画面无缝衔接**，不会有突兀的画风跳变
4. **Seedance 生视频**：豆包 T2V 单段最长 6.5 秒。如果本段需要 12s / 18s，系统拆 chunk，每段尾帧自动作为下段首帧串成长视频

**这是产品最有技术含量的功能**，因为豆包 T2V 单调用只能出 6.5s，要做出连贯长视频必须自己设计**首尾帧链**。我把它做成对用户透明——用户只看到"6.5s / 12s / 18s"三个时长选项，背后是 1/2/3 个 chunks 自动串接。

##### 衍生：AI 生图再渲染（带运镜）

如果用户已经有一张满意的静态图（或本段已经走完上面的 Seedream 单出图路径），可以选"再渲染"把图变成有运镜的视频段——避免"PPT 静帧"。

**运镜由 LLM 为这一镜头单独决策**：综合本镜主体、本镜在段落里的角色（开场 / 推进 / 高潮 / 收尾）、整片节奏 tempo，输出推近 / 拉远 / 上摇 / 下摇 / 横摇左 / 横摇右 / 视差等运镜方式 + 强度等级。**不是用一套粗规则套所有镜头**——开场的"展品揭幕"和高潮的"人物特写"该用什么运镜完全不同，由 LLM 看完镜头本身的内容来决定。

支持的 6 个方向 + 视差在两条渲染路径上都跑得通，**不会因为后台走了哪条渲染路径就退化成静帧**。

### Step 6 · 工作坊编辑

补全完成后，用户进 4 轨时间线：

- **内容轨**：每个 Scene（镜头）一个色块，可拖拽、可点开看细节、可换素材；色块右上角的**匹配度圆点**（绿/橙/灰）让用户一眼看出哪段配得准、哪段还需修补
- **字幕轨**：每段口播 / 字幕的时间窗
- **包装轨**：标题条、贴纸、转场（包装层产物）
- **BGM 轨**：背景音乐 + 音量曲线

#### ⌘K 自然语言编辑（双入口）

用户可以按 ⌘K 唤出对话框，用自然语言改 Plan：

> "把第 2 段缩短一半，加点紧迫感"
>
> "结尾加一个转场到产品特写"
>
> "BGM 在高潮处响一点"

LLM 输出**操作 ops**（修改字段、增删 Scene、换素材等），但不直接落地——前端先 **dry-run 预览**给用户看，用户确认才 apply。每次 apply 生成新 plan_id 入撤销栈，撤回随时可回到任何历史版本。

**作用域规则**：在 Plan 阶段（step2）⌘K 只能改内容轨；在工作坊（step3）⌘K 禁动内容轨，但字幕/转场/BGM/包装全开。**这是为了防止用户在精修阶段误改大结构**。

#### 包装层（Packaging）

LLM 一次性给 6 种风格的包装方案（字幕样式 + 转场 + 封面），每种方案有不同的视觉调性。用户挑一种 apply，所有字幕、转场、封面同步入轨。

也支持**单项手动加**：一段标题条、一个贴纸、一个 CTA 字卡，都能拖到时间线任意位置。

---

### Step 7 · 渲染

用户点"开始渲染"，进度条 6 步：

| 步骤 | 进度 | 做什么 |
|---|---|---|
| 1. 准备 | 8% | 解析 Plan、下载远端 AIGC 视频到本地、生成字卡图 |
| 2. 主轨拼接 | 28% | 各 Scene 按段裁剪 + 归一画幅 + 拼成主视频 |
| 3. AIGC 等待 | 48% | 等待豆包 Seedance 生成完成、串接首尾帧 chunks |
| 4. 包装渲染 | 70% | Remotion 渲染**透明 WebM**（字幕 / 标题条 / 贴纸 / 转场 / 封面） |
| 5. 叠加 | 88% | 透明 WebM overlay 到主轨上 |
| 6. 收尾 | 99% | 输出最终视频 + 自动生成封面图 |

每步前端都有动画 + 实时进度。**关键设计**：包装层走 Remotion 渲染**透明 WebM 而不是直接画在主轨**——这样字幕样式可以独立改、独立换、独立调透明度，不影响主视频。

最终产物：一支 30 秒 9:16 的 mp4 + 一张封面图，用户直接下载。

---

### Step 8 · 跨项目知识沉淀（让产品越用越懂你 · 软性差异化）

用户每完成一次渲染：

1. 系统记录"v0 vs v1 段落 diff"（用户改动了什么）
2. 系统记录所有 NL 编辑记录（用户用 ⌘K 说了什么、撤回了什么）
3. LLM **蒸馏**这些记录，提炼成**用户偏好规则**，5 类范畴：
   - **structure**：段落结构偏好（"climax 段固定拆成两段"）
   - **source**：镜头来源偏好（"高潮段拒绝 AIGC、必用真镜头"）
   - **narration**：文案风格偏好（"句末禁用感叹号"）
   - **pacing**：节奏偏好（"开场不超过 3 秒"）
   - **packaging**：包装层偏好（"封面用大色块红黑配"）

下次新项目生成 Plan 时，最近 10 个已渲染项目的 rules 自动注入 prompt 顶部。**LLM 沿用你过往验证过的偏好**——用得越多越懂你。

撤回的编辑（用户 dry-run 预览后说"不要这个方向"）也会被采集为**负信号**："避免 X"，避免下次再走错路。

## 3 · 功能创新点回顾（2 分钟）

不讲技术栈，只讲**用户能感知到的功能差异化**：

1. **理解优先的样例拆解**：先让 AI 看完整片再切段，6 种叙事模式 + 17 个角色名，对真实视频的适配度远高于"开头-中间-结尾"硬模板
2. **可手动修正的拆解结果**：拆错了，用户当场改，系统记住改动，新版本立即生效
3. **subject_anchors 硬约束**：用户白名单里的物体一个都不会被 LLM "营销词替换"丢掉
4. **素材按内容自动分发 + 分镜匹配度可视**：上传的视频会按内容切成多个分镜、图片走内容理解；Plan 完成的瞬间，每段已被自动配上最匹配的分镜，**同一支视频的不同分镜可以分别去到不同段落**；每个 Scene 卡显示绿/橙/灰匹配度圆点，让用户对"哪段配得准"有直观反馈
5. **节奏画像驱动的结构迁移**：把样例的段落时长比例、节奏密度、情绪峰值时刻直接喂给 LLM，让它在新片对应位置安排能撑起情绪的段落 —— **不抄画面，只抄节奏**，这是结构能真正用上的信号
6. **三路缺口补全**：人工挑 / 字卡 / AIGC 同一交互入口，用户按需选
7. **AIGC 首尾帧链 + 每镜头 LLM 直选运镜**：解决豆包 T2V 单段 6.5s 限制，自动串成长视频画面无缝衔接；"AI 生图再渲染"路径上，每个镜头的运镜（推/拉/平移/视差）由 LLM 综合本镜主体 + 角色 + 整片节奏单独打分，两条渲染路径都跑通同一套运镜，不会退化成 PPT 静帧
8. **LLM anchor + 规则插值的情绪曲线**：把"情绪"这个看不见摸不着的东西，变成 60 点曲线 + 标好峰值位置的可视化产物
9. **dry-run + 撤销栈的 NL 编辑**：用户随便说，先看效果再决定要不要——不会有"AI 自作主张改坏了"的焦虑
10. **跨项目知识沉淀**：用得越多越懂你，新项目自动套用过往偏好

## 4 · 收尾（30 秒）

整个项目的核心思想是把**"看爆款学不会"**这件事的瓶颈从**人的手感**转移到**系统的结构理解能力**——只要 AI 能把样例拆得足够准，把用户意图理解得足够具体，把硬约束写得足够死，就能把"结构迁移"这件事做成一个普通用户能 5 分钟上手的工具。
